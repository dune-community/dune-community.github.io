Detailed Description

template<class OperatorImp, class DiscreteFunctionImp, class SolverImp, class TimeFieldImp = double, RosenbrockTimeStepperMethods method = RosenbrockTimeStepperMethods::GRK4T>
class Dune::GDT::RosenbrockTimeStepper< OperatorImp, DiscreteFunctionImp, SolverImp, TimeFieldImp, method >

Time stepper using Rosenbrock-type methods.

Timestepper using Rosenbrock-type methods methods to solve equations of the form u_t = r * L(u) where u is a discrete function, L an operator acting on u and r a scalar factor (e.g. -1). The specific Rosenbrock-type method can be chosen as the third template argument. If your desired Rosenbrock-type method is not contained in Dune::GDT::TimeStepper::RosenbrockTimeStepperMethods, choose RosenbrockTimeStepperMethods::other and supply matrices A, Gamma (DynamicMatrix< RangeFieldType >) and vectors b_1, b_2 (DynamicVector< RangeFieldType >) and c (DynamicVector< TimeFieldType >) in the constructor. Here, A, Gamma, b_1, b_2 and c form the extended butcher tableau (see https://de.wikipedia.org/wiki/Rosenbrock-Wanner-Verfahren, b_1 and b_2 are the same as for adaptive Runge-Kutta schemes). The default is the GRK4T method.

Template Parameters
OperatorImpType of operator L
DiscreteFunctionImpType of initial values and solution at a fixed time
SolverImpType of solver used for inversion of matrix in each time step.
TimeFieldImpType used for representation of time (default is double)
methodRosenbrock-type method that is used (default is RosenbrockTimeStepperMethods::GRK4T)

Find discrete solution for time point that is closest to t.

The timestepper only stores the solution at discrete time points. This function returns the discrete solution for the stored time point that is closest to the query time t.

Perform one time step and return the estimated optimal time step length for the next time step.

Parameters
dtTime step length.
max_dtMaximal allowed time step length in this time step.
Returns
Estimated optimal time step length for the next time step.
Note
For adaptive methods or if max_dt < dt, dt is just the initial time step length, the actual time step taken may be shorter. To get the actual time step length you need to compare current_time() before and after calling step().
This method has to increase the current time by the actual time step length taken, otherwise solve will never finish execution (if current_time is not increased) or give wrong results (if current time is increased by a wrong dt).
