Detailed Description

template<class LeftType, class RightType, Combination comb>
class Dune::XT::Functions::internal::Combined< LeftType, RightType, comb >

Generic combined function.

This class combines two given functions of type LeftType and RightType using the given combination Combination. This class (and any derived class, like Difference, Sum or Product) can be used in two was:

  • You can pass references of the left and right operand to this class. This is done for instance when calling operator+, operator- or operator* on any function deriving from LocalizableFunctionInterface:
    typedef Functions::Constant< ..., double, 2, double 1 > ConstantType;
    ConstantType one(1);
    ConstantType two(2);
    // the following code
    auto difference = one - two;
    // is equivalent to
    Difference< ConstantType, ConstantType > difference(one, two);
    // and
    internal::Combined< ConstantType, ConstantType, Combination::difference > difference(one, tow);
    In this situation you are responsible to ensure that the arguments given are valid throughout the lifetime of this class. The following will lead to a segfault:
    typedef Functions::Constant< ..., double, 2, double 1 > ConstantType;
    Difference< ConstantType, ConstantType > stupid_difference()
    {
    ConstantType one(1);
    ConstantType two(2);
    return one - two;
    }
  • You can pass shared_ptr of the left and right operands to this class. In this case the following is valid:
    typedef Functions::Constant< ..., double, 2, double 1 > ConstantType;
    Difference< ConstantType, ConstantType > stupid_difference()
    {
    auto one = std::make_shared< ConstantType >(1);
    auto two = std::make_shared< ConstantType >(2);
    return Difference< ConstantType, ConstantType >(one, two)
    }
Note
Most likely you do not want to use this class diretly, but one of Difference, Sum or Product.
